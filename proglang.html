Chapter 6: Programming Languages

Term: Definition:
Machine Language: Definition: Programs for modern computers that consist of sequences of instructions that are encoded as numeric digits
Debugging: Definition: Errors that must be located and corrected
Assembler: Definition: Programs developed to convert mnemonic expressions into machine language instructions
Assembly Language: Definition: A mnemonic system for representing programs collectively
Translator: Definition: Translates programs expressed in high-level primitives into machine-language programs
Compiler: Definition: Translation program that compiles several machine instructions into short sequences
Interpreter: Definition: Alternative to translators except they execute the instructions as they were translated instead of recording the translated version for future use
Cross-Platform Software: Definition: Software that is independent of an operating system's design and the machine's hardware design/ executable throughout a network
Natural Language: Definition: Such as English, German, and Latin
Formal Language: Definition: Programming languages
Machine Independence: Definition: Do not rely on the characteristics of a particular machine
High-Level Programming Language: Definition: Provide more abstractions for the programmer and make it easier for people to read and write a program
Object-Oriented Programming (OOP): Definition: Software system is viewed as a collection of units called objects, performing the actions that are immediately related to itself and requesting actions of other objects
Scripting Language: Definition: The collection of languages that are a subset of the imperative programming languages
Data Types: Definition: Ecompasses both the manner in which the data item is encoded and the operations that can be performed on that data
Integer: Definition: Numeric data consisting of whole numbers
Float: Definition: Numeric data that might contain values other than whole numbers
String: Definition: A sequence of characters used for storing text
Primitive Data Type: Definition: Data types that are included as primitives in a programming language, such as (int) for integer and (char) for character
Data Structure: Definition: The conceptual shape or arrangement of data, often associated with variables in a program
Python Collection Data Structures: Definition: Lists, Tuples, Sets, and Dictionaries
List: Definition: Used to store multiple items in a single variable, one of 4 built-in data types in Python used to store collections of data, created using square brackets
Tuple: Definition: Used to store multiple items in a single variable, one of 4 built-in data types in Python used to store collections of data, written with round brackets
Set: Definition: Used to store multiple items in a single variable, one of 4 built-in data types in Python used to store collections of data, written with curly brackets
Dictionary: Definition: Used to store data values in key: value pairs, a collection which is ordered, changeable, and do not allow duplicates
Function: Definition: A set of instructions for performing a task that can be used as an abstract tool by other program units
Local Variable: Definition: A variable declared within a function, can be referenced only with that function
Global Variable: Definition: A variable whose scopes are not restricted to a particular part of a program
Source Program: Definition: The program in its original form
Object Program: Definition: The translated version of the source program
